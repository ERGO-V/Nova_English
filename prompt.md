## 项目名称：NovaEnglish（Flutter背单词应用）

### 一、技术栈与平台
- **框架**：Flutter（稳定版）
- **目标平台**：Android + Web（优先保证两者体验一致）
- **本地数据库**：sqflite（存储所有数据）
- **状态管理**：推荐Provider或Riverpod（灵活选择）
- **主题**：暗色调为主（背景深色，卡片高对比）
- **无后端**：所有数据存储在本地，通过JSON文件导入导出实现跨设备同步

### 二、核心功能模块

#### 1. 数据模型设计
按照以下表结构创建数据库（表名、字段名可酌情调整，但需满足功能）：
- **builtin_dict**：内置词典（四级、六级、考研、雅思、托福）  
  `id`（主键）, `word`（英文）, `meaning`（中文释义）, `source`（词典来源：cet4/cet6/kaoyan/ielts/toefl）
- **oxford_dict**：牛津词典（仅用于自定义单词的自动填充，不可学习）  
  `id`, `word`, `meaning`（基本释义）
- **custom_dict**：用户自定义词典  
  `id`, `name`（词典名称）
- **unit**：单元（属于某个自定义词典）  
  `id`, `dict_id`（关联custom_dict.id）, `name`
- **custom_word**：自定义单词（存储在某个单元下）  
  `id`, `unit_id`, `word`, `meaning`（用户可手动修改或自动填充）
- **learning_progress**：记录所有可学习单词的学习状态（包括内置单词和自定义单词，但自定义单词仅记录学习次数和错误次数，不参与复习算法）  
  `id`, `word_id`（关联内置词典或自定义单词的id）, `word_type`（'builtin' 或 'custom'）, `error_count`（累计错误次数，只增不减）, `last_review_time`（最后复习/学习时间，Unix时间戳）, `next_review_time`（下次复习时间戳）, `consecutive_correct`（连续正确次数，0-3）, `is_removed`（是否已从复习池移除，0/1）

> 说明：自定义单词虽然不参与复习，但仍记录错误次数供用户查看，且学习时一次性学完单元所有词，交互同卡片形式。

#### 2. 学习模块（针对内置词典）
- **入口**：主界面（学习Tab）提供下拉选择框，选择要学习的内置词典（四级/六级/考研/雅思/托福）。
- **学习机制**：
  - 点击“开始学习”按钮，从当前词典中**随机抽取20个未达到“已掌握”状态的单词**（即仍在复习池中的单词，is_removed=0）。如果不足20个，则取全部。
  - 学习模式采用**卡片形式**：
    - 卡片只显示英文，不显示中文。
    - 底部两个按钮：“认识”和“不认识”（或“记错”）。
    - 点击“认识”或“不认识”后，**才显示中文释义**，同时弹出确认按钮（或直接记录结果）。
    - 记录结果：
      - 点击“认识”：连续正确次数+1，错误次数不变，根据算法更新下次复习时间。如果连续正确次数达到3，则将该单词从复习池移除（is_removed=1）。
      - 点击“不认识”：错误次数+1，连续正确次数归零，下次复习时间 = 当前时间 + 1天（且保证单词仍在复习池中，is_removed=0）。
    - 用户如果发现记错（比如点击了认识但实际错误），可以在卡片上提供一个“记错了”按钮（或重新选择），此时触发“不认识”逻辑，且该单词**立即再次进入本次学习队列**，让用户再次学习（可设置最多重学次数或直接重新插入队列尾部）。
  - 学习结束后，显示本轮学习统计：本次学习单词总数，正确/错误次数等。
- **复习模块**（集成在学习页面）：
  - 单独一个按钮“复习”，点击后从复习池中抽取**20个已到复习时间且未移除的单词**（next_review_time <= 当前时间）。
  - 抽取排序规则：按`next_review_time`升序，相同时间按`error_count`降序（错误多的优先）。
  - 复习交互与学习模式完全相同（卡片显示英文，点击认识/不认识后显示释义，并更新复习状态）。
  - 复习池单词在连续正确三次后自动移除。

#### 3. 自定义词典模块
- **我的词典Tab**：底部导航栏第二个入口。
- **界面结构**：
  - 第一级：展示所有自定义词典列表。底部固定一个“添加词典”按钮，点击后弹出对话框输入词典名称，保存后刷新列表。
  - 点击某个词典 → 进入该词典的**单元列表**。底部有“添加单元”按钮，点击后输入单元名称，保存。
  - 每个单元卡片右侧包含：
    - **下拉图标**：点击后展开显示该单元的**前几个单词**（例如最多5个），单词列表可横向滚动。展开时仅预览单词英文，不显示释义。
    - **学习图标**（如📖）：点击进入该单元的**学习页面**。
    - **修改图标**（如✏️）：点击进入**单元编辑页面**，纵向排列展示所有单词（每个单词显示英文+释义），每个单词右侧有修改和删除按钮。页面右上角有保存按钮，用于保存对单词的修改（新增、修改、删除）。
- **学习自定义单元**：
  - 一次性学习该单元内**所有单词**（不限于20个），顺序随机。
  - 交互与内置词典学习模式相同：卡片只显示英文，点击认识/不认识后显示中文，并记录错误次数（错误次数累加，不进入复习池，但保存在learning_progress中）。
  - 学习过程中，如果用户点击不认识或记错，单词会再次出现（同内置词典规则），直到用户正确认识为止？规则可设为：连续错误后，单词会在本轮中重复出现，直到用户正确认识一次才从本次队列移除（避免卡死）。具体由codex实现时设计合理逻辑。
  - 学习结束后，展示本轮学习统计，包括错误次数最多的单词等。
- **添加单词功能**：
  - 在单元编辑页面（或单元详情页）提供“添加单词”按钮。
  - 添加单词时，输入英文单词，系统实时在`oxford_dict`表中进行**模糊搜索**（如`LIKE '%输入%'`），展示匹配项列表，用户选择后自动填充中文释义。若未找到，则允许用户手动输入释义。
  - 保存后，单词存入`custom_word`表，并关联当前单元。

#### 4. 导出/导入功能
- **导出**：
  - 在设置页面提供“导出数据”按钮。
  - 导出内容包括：
    - 内置词典的学习进度（learning_progress中word_type='builtin'的记录）
    - 自定义词典的所有数据（custom_dict, unit, custom_word表）
  - 导出格式为JSON文件，结构自定但需清晰包含版本号，便于未来升级。
  - 导出文件保存到本地（Android通过文件管理器，Web通过浏览器下载）。
- **导入**：
  - 在设置页面提供“导入数据”按钮，选择JSON文件。
  - 解析后，**以导入文件为准**合并到当前数据库：
    - 对于自定义词典：直接覆盖同名词典（或根据id判断，若id冲突则覆盖），需注意关联关系。
    - 对于学习进度：根据单词id和word_type，覆盖现有进度记录。
  - 导入前建议备份当前数据库（可提示用户）。

  - 昵称编辑：输入框，保存后更新显示。

### 四、复习算法细节（明确）
- 每个单词维护以下字段：
  - `error_count`：累计错误次数（只增不减）
  - `consecutive_correct`：连续正确次数（0-3）
  - `next_review_time`：下次复习时间戳
- 学习/复习时：
  - 点击“认识”：
    - 若`consecutive_correct` < 3，则`consecutive_correct++`
    - 如果`consecutive_correct`达到3，则设置`is_removed = 1`（从复习池移除）
    - 更新`next_review_time` = 当前时间 + (2^`consecutive_correct`) 天（即1,2,4天）
  - 点击“不认识”：
    - `error_count++`
    - `consecutive_correct = 0`
    - `next_review_time = 当前时间 + 1天`
    - `is_removed = 0`（确保单词在复习池中）
- 复习池查询条件：`is_removed = 0 AND next_review_time <= 当前时间`
- 单词在达到连续正确3次后，未来如果再次学习（如用户主动重新学习）时出错，应重置`is_removed=0`，重新进入复习流程。
